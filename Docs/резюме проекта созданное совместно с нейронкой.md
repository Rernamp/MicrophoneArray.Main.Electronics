# Резюме проекта: 4-канальная микрофонная решетка на CH32V307VCT6

## 1. Общая архитектура

| Компонент | Решение |
|-----------|---------|
| **MCU** | CH32V307VCT6 (отладочная плата) |
| **Микрофоны** | 4 × PDM (INMP441 или аналогичные) |
| **Интерфейс** | 2 × I2S (один Master, один Slave) |
| **Тактирование** | TIM-делитель (BCLK → CLK) |
| **Синхронизация** | Master генерирует BCLK и WS; Slave тактируется от них |
| **Питание** | Отдельная малошумящая LDO (5В → 3.3В) |
| **USB** | Передача аудиоданных на ПК |

---

## 2. Решенные ключевые задачи

### 2.1. Синхронизация I2S (Master + Slave)
- **Проблема:** Как запустить два I2S строго одновременно?
- **Решение:** Включение Slave **до** Master. Первый фронт BCLK, сгенерированный Master, одновременно достигает обоих I2S.
- **Статус:** ✅ Решено (подтверждено даташитом CH32V307).

### 2.2. Тактирование микрофонов (TIM-делитель)
- **Проблема:** PDM требует CLK = BCLK/2.
- **Решение:** Внешний TIM делит BCLK (6.144 МГц) на 2, подает CLK (3.072 МГц) на все микрофоны.
- **Статус:** ✅ Решено (требуется проверка задержки таймера).

### 2.3. Прием данных с 4 микрофонов
- **Проблема:** У I2S только 2 DATA-линии (по одной на экземпляр).
- **Решение:** По 2 микрофона на каждой DATA-линии (один настроен на левый канал, другой на правый).
- **Статус:** ✅ Решено.

### 2.4. Синхронизация с USB (использование MCK)
- **Проблема:** Как точно узнать, когда семпл готов для отправки по USB?
- **Решение:** TIM считает импульсы MCK (12.288 МГц), прерывание по переполнению = готовность семпла.
- **Статус:** ✅ Решено.

### 2.5. Питание микрофонов
- **Проблема:** Цифровые шумы от MCU могут ухудшить качество звука.
- **Решение:** Отдельная малошумящая LDO (например, LP2985 или TPS7A20) от 5В USB.
- **Статус:** ✅ Решено (требуется выбор конкретной LDO).

## 3. Частотный план

| Параметр | Значение | Формула |
|----------|----------|---------|
| Частота дискретизации (Fs) | 48 кГц | Целевая |
| Частота WS (LRCK) | 48 кГц | Fs |
| BCLK (I2S) | 6.144 МГц | Fs × 64 × 2 (стерео, 32 бита) |
| CLK (микрофоны) | 3.072 МГц | BCLK / 2 |
| MCK | 12.288 МГц | Fs × 256 |
| USB-передача | 48 кГц × 4 канала × 16 бит = 3.072 Мбит/с | — |

---

## 4. Программная обработка (критический этап)

1.  **Прием через DMA:** Два канала DMA (для I2S1 и I2S2).
2.  **Деинтерливинг:** Распаковка битов из стереопотока в 4 отдельных потока.
3.  **PDM → PCM:** Программный CIC-фильтр (или библиотека от WCH).
4.  **USB-отправка:** Аудиопоток в формате 16-бит/48 кГц (или 24 бит, если позволяет USB).

**Рекомендация:** Начните с 16-битного режима I2S (DATLEN=16, CHLEN=16), чтобы упростить деинтерливинг.

---

## 5. Рекомендации по разводке печатной платы (Shield)

1.  **Питание:**
    *   LDO — как можно ближе к микрофонам.
    *   Керамические конденсаторы: 10 мкФ + 100 нФ на входе LDO; 1 мкФ + 100 нФ на выходе (у каждого микрофона).
    *   Разделяйте аналоговую (микрофоны) и цифровую (MCU) земли.

2.  **Тактовые линии (BCLK, CLK):**
    *   Максимально короткие дорожки.
    *   Согласование импеданса (резисторы 22-33 Ом последовательно с CLK).
    *   Не размещайте CLK рядом с DATA-линиями во избежание наводок.

3.  **DATA-линии:**
    *   Подтяжка к питанию (10 кОм) для обеспечения логического уровня в состоянии High-Z.

4.  **Разъемы:**
    *   Гребенка для подключения внешней платы с микрофонами (CLK, DATA1, DATA2, VDD, GND).
    *   Дополнительные пины для BCLK, WS, MCK (для отладки).

## 6. Найденные противоречия и способы их решения

| Противоречие | Решение |
|--------------|---------|
| I2S читает только по одному фронту | Использование BCLK = 2×CLK |
| Таймер-делитель вносит задержку | Выбор микрофона с большим запасом по таймингам; снижение частоты Fs (например, до 44.1 кГц) |
| Slave может начать прием со сдвигом | Включение Slave → Master |
| Шумы по питанию | Отдельная LDO + фильтрация |
| MCK не нужен для микрофонов | Использовать MCK для точной синхронизации USB |

---

## 7. Что требует дополнительной проверки

1.  **Задержка таймера-делителя:** Измерьте осциллографом задержку между BCLK и CLK при работе на 48 кГц.
2.  **PDM-фильтр:** Оцените загрузку CPU при фильтрации 4 каналов на частоте 48 кГц.
3.  **USB-пропускная способность:** Убедитесь, что USB 2.0 Full Speed (12 Мбит/с) справляется с потоком 4×48×16 = 3.072 Мбит/с.

**Проект технически состоятелен.** Все ключевые архитектурные решения приняты и обоснованы:

*   Синхронизация I2S — через Master/Slave с включением Slave до Master.
*   Тактирование микрофонов — через TIM-делитель.
*   Прием данных — по 2 микрофона на DATA-линию.
*   Синхронизация USB — через TIM и MCK.
